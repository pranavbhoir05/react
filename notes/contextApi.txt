Context API ‚Äì Quick Notes

Definition :
React feature for sharing state across components without prop drilling.

Why it exists :
To avoid passing props through multiple layers just to reach deep children.

Core Pieces
1. Context ‚Üí created with createContext().
2. Provider ‚Üí supplies the value to the component tree.
3. useContext() ‚Üí consumes the value in any child component.

Flow : create context ‚Üí wrap components with Provider ‚Üí read value using useContext

Best Use Cases :
Auth user
Theme (light/dark)
Language
Cart/global UI state

When NOT to use :
Large or frequently changing state ‚Üí causes many re-renders.
Complex logic ‚Üí prefer Redux/Zustand/Recoil.

Pros :
Simple
No external library
Eliminates prop drilling

Cons:
Can impact performance if overused
Harder debugging than dedicated state managers

Rule of Thumb :
Small to medium shared state ‚Üí Context API
Large/global dynamic state ‚Üí dedicated state manager

‚úÖ Scenario: Simple project with theme toggle
You have components:

App

____________________________________________________________________________________

‚ùå Without Context API (prop drilling)
You keep theme in App:

function App() {
  const [theme, setTheme] = useState("light");
  return <Header theme={theme} setTheme={setTheme} />;
}
But Home and Footer also need theme.

So you must do this:

App ‚Üí Header (theme)
App ‚Üí Home (theme)
App ‚Üí Footer (theme)
But imagine Header has 3 nested components inside:

App ‚Üí Header ‚Üí HeaderTop ‚Üí HeaderButtons
If HeaderButtons needs theme, you must drill:

Header passes theme to HeaderTop
HeaderTop passes theme to HeaderButtons
Even if Header & HeaderTop DON‚ÄôT use theme at all.

This becomes messy fast.
____________________________________________________________________________________

‚úÖ With Context API (no drilling)
You simply wrap once:

<ThemeContext.Provider value={{ theme, setTheme }}>
  <AppContent />
</ThemeContext.Provider>
Now ANY component can directly access theme:

const { theme } = useContext(ThemeContext);
No passing.
No drilling.
No props.
Clean.

üî• The simplest working example

* App.jsx
